# 🚀 Project Design: Oryzo Landing Page (Vite)

## 🏗️ Architecture Overview
The project has been migrated from a single HTML file to a professional **Vite-based** modular architecture. This ensures better maintainability, faster development (HMR), and optimized production builds.

### Tech Stack
- **Build Tool**: Vite 5.x
- **3D Engine**: Three.js r128+
- **Animation**: GSAP 3.12+ (ScrollTrigger)
- **Styling**: Vanilla CSS
- **Language**: ES Modules (Vanilla JS)

---

## 📁 File Structure & Responsibilities

### `index.html`
The main entry point and template. It contains the HTML structure for all 9 sections (Hero, Reveal, Features, Benchmark, etc.) and hooks into `src/main.js`.

### `src/main.js`
The central orchestrator.
- Initializes all sub-modules.
- Handles global events (mouse move, resize).
- Manages the Canvas 2D **Film Grain** animation.
- Manages the DOM-based **Light Leak** system (3-layer trailing effect).
- Runs the main `requestAnimationFrame` loop.

### `src/three-setup.js`
Encapsulates all Three.js logic.
- **Scene & Camera**: Sets up a `PerspectiveCamera` and `FogExp2`.
- **Renderer**: High-performance WebGL renderer with pixel ratio capping.
- **Shaders**: Implements a robust **ShaderMaterial** for the coaster.
  - **Rim Lighting**: Calculated via dot product in GLSL for a cinematic glow.
  - **Procedural Cork**: (Future enhancement) Logic to generate texture via noise.
- **Robust Loading**: Includes fallback shaders to prevent crashes if splitting fails.

### `src/scroll.js`
Handles custom scroll momentum and lerping.
- **Lerp System**: Smooths out scroll delta for a "premium" feel.
- **Proxying**: Connects our custom scroll position to GSAP's `ScrollTrigger`.

### `src/animations.js`
Contains all GSAP-based visual effects.
- **ScrollTrigger**: Syncs camera movement and coaster rotation to scroll progress.
- **Text Reveal**: Word-by-word opacity animation for the Reveal section.
- **Benchmark Bars**: Intersection-triggered width animations.
- **Section Entrances**: Staggered fades and transforms for a cinematic feel.

### `src/shaders/`
Dedicated GLSL files to keep code clean.
- `rimLight.glsl`: Combined vertex/fragment shader for the main object.
- `filmGrain.glsl`: Post-processing style grain (available for extension).
- `lightLeak.glsl`: Shader-based light leak (available for extension).

### `src/utils/`
- `constants.js`: Centralized colors and configuration values.
- `helpers.js`: Reusable utility functions (lerp, debounce, mouse mapping).

---

## 🎨 Design Philosophy
1. **Deadpan Aesthetic**: Minimalist, high-contrast (#080808 and #C8FF00), and brutalist typography (Bebas Neue).
2. **Cinematic Depth**: Achieved through animated film grain, light leaks, and exponential fog.
3. **Tactile 3D**: The coaster isn't just a static object; it reacts to mouse movement and scroll, with lighting that feels physical.

---

## 🔧 Optimization & Performance
- **Manual Chunking**: Three.js and GSAP are split into separate vendor chunks for better caching.
- **Terser Minification**: Removes console logs and optimizes code size for production.
- **Target `esnext`**: Leverages modern JS features for smaller bundles.
- **Asset Handling**: Shaders are imported as raw strings via `vite-plugin-string`, reducing network requests.
